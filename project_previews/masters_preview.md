# Masters Tournament Intelligence

### Interactive Tournament Analytics App

---

I built Masters Tournament Intelligence as an earlier analytics project focused on data storytelling, pipeline validation, and front-end reporting. It shows how sports event data can be structured into interactive scorecards, leaderboard views, and tournament narratives.

---

## Problem

Tournament coverage is often reduced to raw scores and a leaderboard. I wanted this project to build a more navigable analytical surface: scorecards, player views, course context, leaderboard movement, and round-by-round exploration.

---

## Data and workflow

The repo contains a pipeline for pulling and transforming tournament and scoring data into generated JSON artifacts used by the React application. I put the emphasis on validating those artifacts before display.

---

## System

The project includes:

- Vite and React application.
- TypeScript pipeline scripts.
- Generated JSON artifacts.
- Scorecard and course-data validation.
- Recharts-based visualizations.
- Tailwind styling.
- Vercel deployment configuration.

---

## Evaluation and reliability

The reliability signal here is the validation-first pipeline and generated-artifact approach. I think of this as a front-end analytics product with disciplined data preparation rather than a large-scale backend system.
